# مرکز اهدا پلاسما نوین پلاسما پورا دارو — سامانه‌ی CRM کامل

فقط ۲ فایل لازمه: **server.js** (همه‌چیز داخلشه) و **package.json**.
مراحل دیپلوی روی Railway دقیقاً همون مراحل قبلیه: آپلود این ۲ فایل توی یه ریپوی گیت‌هاب → اتصال به Railway → تنظیم Variables → فعال کردن Volume روی مسیر `/app/data` → گرفتن لینک.

## نام کاربری اولیه
- `admin` / `admin1234` (یا هرچی که در Variables با `ADMIN_USERNAME`/`ADMIN_PASSWORD` ست کرده باشید) — سیستم مجبورتون می‌کنه بلافاصله عوضش کنید.

## این‌بار چی درست/اضافه شد

**رفع مشکل «مسدود است»**: توی نسخه‌ی قبل، وقتی یه اهداکننده مسدود می‌شد، هیچ راهی برای رفع مسدودی نبود. حالا:
- توی صفحه‌ی جزئیات هر اهداکننده، اگه وضعیتش «مسدود شده» باشه، دکمه‌ی **«رفع مسدودی»** ظاهر می‌شه.
- دکمه‌ی **«حذف کامل این اهداکننده»** هم اضافه شد (فقط برای مدیر، با تاییدیه‌ی دوباره چون قابل بازگشت نیست).
- دکمه‌ی **«حذف نظرسنجی»** هم اضافه شد تا بشه نظرسنجی یه فرد رو پاک کرد و بذارید دوباره پرش کنه (مثلاً برای تست).

**اتصال واقعی به sms.ir**: حالا پیامک‌ها واقعاً از طریق وب‌سرویس sms.ir ارسال می‌شن (نه فقط نمایش داخل پنل). فقط کافیه این ۲ متغیر رو توی Railway → Variables اضافه کنید:
- `SMS_IR_API_KEY` — از پنل sms.ir، بخش «وب‌سرویس» یا «REST API» بگیرید.
- `SMS_IR_LINE_NUMBER` — شماره خط اختصاصی‌تون در sms.ir (فقط عدد).

اگه این دو مقدار رو نذارید، سیستم مثل قبل کد OTP و پیام‌ها رو مستقیم داخل پنل نشون می‌ده (حالت دمو) تا بتونید بدون هزینه تستش کنید.

**طراحی کاملاً جدید**: فونت فارسی واقعی (Vazirmatn از گوگل فونت)، آیکون‌های حرفه‌ای به‌جای اموجی، هدر گرادیانی برای پنل مدیریت، کارت‌ها با سایه و انیمیشن ظریف، دکمه‌های گرادیانی، navigation-tile برای دسترسی سریع به بخش‌های مختلف پنل.

## نکته‌ی مهم درباره‌ی Volume (اگه از قبل دارید نیازی به کار جدید نیست)
اگه از دفعه‌ی قبل Volume روی `/app/data` رو فعال کرده بودید، اطلاعات قبلی‌تون (اهداکنندگان، کارکنان) دست‌نخورده باقی می‌مونه — فقط کد جدید جایگزین کد قدیم می‌شه.

## مراحل دیپلوی (یادآوری کامل)

1. توی گیت‌هاب، فایل `server.js` قدیمی رو با این نسخه‌ی جدید جایگزین کنید (ویرایش فایل → پاک کردن محتوا → پیست محتوای جدید → Commit).
2. Railway خودکار دوباره دیپلوی می‌کنه.
3. برای فعال کردن پیامک واقعی، به Variables برید و `SMS_IR_API_KEY` و `SMS_IR_LINE_NUMBER` رو اضافه کنید.
4. صفحه رو رفرش کنید و تست کنید.

## عیب‌یابی

- **پیامک نمیاد ولی کد باید بره**: مطمئن بشید `SMS_IR_API_KEY` و `SMS_IR_LINE_NUMBER` هر دو درست وارد شدن (بدون فاصله‌ی اضافه). اگه خط شما هنوز خدماتی/تایید نشده باشه، ممکنه sms.ir ارسال رو رد کنه — از پنل sms.ir وضعیت خط رو چک کنید.
- **اهداکننده هنوز مسدوده**: وارد جزئیات همون فرد بشید، دکمه‌ی «رفع مسدودی» رو بزنید. اگه دکمه رو نمی‌بینید، مطمئن بشید با حساب مدیر (نه کارمند عادی) وارد شدید.
- **می‌خوام داده‌های تست رو پاک کنم**: از جزئیات هر اهداکننده، «حذف کامل این اهداکننده» رو بزنید.


## چیزهایی که اضافه/اصلاح شد طبق سند شما

**ثبت‌نام و ورود**
- ورود اهداکننده با کد پیامکی یک‌بارمصرف (OTP)، نه دیگه با شماره‌ی ساده.
- ثبت‌نام فقط نام، نام‌خانوادگی، موبایل، سن.

**نظرسنجی اولین مراجعه**
- فقط یک‌بار (نه هر بار). شامل حراست، پذیرش، پزشک، پرستار، سایر کارکنان، نظم/سرعت، نظافت، آرامش محیط، رضایت کلی — هرکدوم ۱ تا ۵.
- نحوه‌ی آشنایی با مرکز (روابط عمومی، اینستاگرام، معرفی دوستان، تبلیغات محیطی، جستجوی اینترنتی، مراجعه‌ی قبلی، سایر).
- کادر پیشنهاد/انتقاد آزاد.
- بعد از ثبت، پیام دقیقاً طبق متن شما نمایش داده می‌شه (۷ تا ۱۰ روز، بدون امکان رزرو نوبت).

**فعال‌سازی پنل**: مدیر نتیجه‌ی آزمایش رو تایید می‌کنه → پنل فعال می‌شه → پیامک (طبق قالب قابل‌ویرایش) ارسال می‌شه.

**نوبت‌دهی**: دو حالت خودکار (ظرفیت هر ساعت، نمایش زمان‌های خالی) و دستی (درخواست کاربر → تایید یا پیشنهاد زمان دیگر توسط مدیر) — از تنظیمات قابل تعویضه.

**ثبت نتیجه‌ی مراجعه**: مراجعه کرد / اهدا انجام شد / اهدا انجام نشد (+ علت).

**پیگیری پس از اهدا**: دقیقاً ۵ سوالی که خواستید (وضعیت عمومی، سرگیجه، مشکل محل تزریق، رضایت، آمادگی برای مراجعه‌ی بعد) + هشدار خودکار به مدیر در صورت علائم/نارضایتی.

**پنل مدیریت**: اهداکنندگان، نوبت‌ها، ظرفیت هر ساعت، حالت نوبت‌دهی، ثبت نتیجه‌ی اهدا، گزارش آماری.

**داشبورد روابط عمومی**: میانگین رضایت هر واحد (نمودار میله‌ای بدون نیاز به کتابخونه‌ی خارجی)، نظافت/محیط، نحوه‌ی آشنایی (تعداد و درصد)، روند زمانی، فهرست پیشنهادها/انتقادات. + دکمه‌ی «نسخه‌ی قابل چاپ» که با Ctrl+P مرورگر مستقیم تبدیل به PDF می‌شه.

**تنظیمات کامل**: فاصله‌ی اهدا، روزها/دفعات پیگیری، ظرفیت هر ساعت، ساعت پذیرش، روزهای کاری، تعطیلات رسمی/اختصاصی، حالت نوبت‌دهی، متن تمام پیامک‌ها.

**پنل کاربر**: وضعیت پرونده، زمان نوبت، تغییر/لغو نوبت، تکمیل فرم پیگیری، تاریخچه، تعداد دفعات اهدا.

**پنل مدیر**: تعریف مدیر/کارمند، سطح دسترسی (admin/staff)، مسدودسازی، حذف، لاگ فعالیت، خروجی CSV (باز می‌شه در اکسل)، جستجوی سریع با نام یا موبایل.

**سیستم پیگیری هوشمند CRM**:
- یادداشت‌های داخلی (فقط کارکنان می‌بینن، هیچ‌وقت برای اهداکننده نمایش داده نمی‌شه) با نام ثبت‌کننده و تاریخ، حذف فقط با دسترسی مدیر.
- یادآورهای پیگیری با تاریخ سررسید، نمایش در داشبورد.
- وضعیت‌های ۱۰گانه‌ی دقیقاً طبق سند شما (ثبت‌نام اولیه تا مسدود شده).
- تایم‌لاین کامل تمام رویدادهای هر فرد.
- هشدارهای هوشمند: عدم مراجعه‌ی طولانی، لغوهای متوالی، علائم غیرطبیعی، درخواست تماس، آزمایش ثبت‌نشده.
- داشبورد CRM با شاخص‌های کامل.
- مرکز اعلان‌ها (Notification Center) که همه‌ی کارهای در انتظار اقدام رو یک‌جا نشون می‌ده.

## دو مورد که نیاز به سرویس بیرونی دارن (خارج از توان یک فایل کد)

1. **ارسال پیامک واقعی**: کد فعلاً پیام‌ها رو داخل خود پنل نشون می‌ده (نه پیامک واقعی). اگه یه متغیر `SMS_WEBHOOK_URL` در Variables ست کنید (آدرس API سرویس پیامکی مثل کاوه‌نگار)، سرور خودکار بهش POST می‌فرسته. اتصال دقیق به API یه سرویس خاص نیاز به مستندات همون سرویس داره — بگید کدوم سرویس رو می‌خواید تا کدش رو دقیق بنویسم.
2. **PDF واقعی**: به‌جاش صفحه‌ی چاپ‌پذیر (Ctrl+P → Save as PDF) گذاشتم؛ قابل‌اطمینان‌تر از افزودن کتابخونه‌ی سنگین PDF.

## نکته درباره‌ی کد ملی
در سند شما بخش «ثبت‌نام اولیه» فقط نام/نام‌خانوادگی/موبایل/سن رو خواسته بود، ولی در بخش جستجو به «کد ملی» هم اشاره شده. چون در ثبت‌نام نگرفتیمش، فعلاً جستجو فقط با نام/موبایل کار می‌کنه. اگه بخواید کد ملی هم در ثبت‌نام اضافه بشه، بگید تا اضافه‌ش کنم.
